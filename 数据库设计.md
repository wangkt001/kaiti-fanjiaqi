# "豫见好物"河南文创产品销售平台 - 数据库设计文档

## 1. 数据库设计说明

### 1.1 数据库版本

- MySQL 8.0+

### 1.2 字符集

- 默认字符集：utf8mb4
- 排序规则：utf8mb4_unicode_ci

### 1.3 存储引擎

- InnoDB（支持事务和外键）

### 1.4 命名规范

- 表名：小写字母 + 下划线（复数形式）
- 字段名：小写字母 + 下划线
- 主键：id（BIGINT 自增）
- 外键：关联表名单数形式\_id
- 时间字段：xxx_at（DATETIME 类型）

## 5. 数据库优化建议

### 5.1 索引优化

1. **复合索引**：
   - products: (category_id, status, created_at) - 商品列表查询
   - products: (seller_id, status) - 卖家商品管理
   - orders: (buyer_id, status, created_at) - 买家订单查询
   - orders: (seller_id, status, created_at) - 卖家订单查询
   - reviews: (product_id, status, created_at) - 商品评价查询

2. **全文索引**（MySQL 5.7+）：
   - products: (name, description) - 商品搜索
   - cultural_contents: (title, content) - 资讯搜索

### 5.2 分区策略

对于数据量较大的表，可以考虑分区：

1. **orders 表**：按创建时间进行 RANGE 分区（按月或按年）
2. **system_logs 表**：按创建时间进行 RANGE 分区（按月）
3. **reviews 表**：按创建时间进行 RANGE 分区（按年）

### 5.3 读写分离

- **主库**：处理写操作（INSERT、UPDATE、DELETE）
- **从库**：处理读操作（SELECT）
- **配置**：使用 MyBatis-Plus 的多数据源配置

### 5.4 缓存策略

使用 Redis 缓存以下热点数据：

1. **商品信息**：缓存商品详情（30 分钟）
2. **用户信息**：缓存用户基本信息（1 小时）
3. **分类和标签**：永久缓存，更新时失效
4. **系统设置**：永久缓存，更新时失效
5. **计数器**：浏览量、点赞数等（实时写入 Redis，定时同步到 MySQL）

## 7. 数据库安全

### 7.1 用户权限管理

创建数据库用户并分配权限：

```sql
-- 创建应用用户（仅应用访问权限）
CREATE USER 'yujian_app'@'%' IDENTIFIED BY 'your_password';
GRANT SELECT, INSERT, UPDATE, DELETE ON yujianhaowu.* TO 'yujian_app'@'%';

-- 创建只读用户（报表查询用）
CREATE USER 'yujian_readonly'@'%' IDENTIFIED BY 'your_password';
GRANT SELECT ON yujianhaowu.* TO 'yujian_readonly'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 7.2 敏感数据加密

1. **密码**：使用 BCrypt 加密
2. **手机号**：可选加密存储，使用时解密
3. **身份证号**：加密存储（如需要）

### 7.3 数据备份策略

1. **全量备份**：每天凌晨 2 点全量备份
2. **增量备份**：每小时增量备份
3. **binlog 备份**：开启 binlog，实时备份
